HTML

  • The structure of code
  • The head element contains info for the webpage
  • The body element represents visible content shown to the user
  • HTML stands for HyperText Markup Language
  • A parent element, such as "body" contains child elements, such as "ul" unorganized list
  • UTF-8 stands for Unicode Transformation Format - 8 bits, and can encode all code points in unicode

CSS

  • The design and style of code
  • A margin indicates how much space we want around the outside of an element
  • Padding determines how much space we want around the content inside an element
  • CSS stands for Cascading Style Sheets
  • rel defines the relationship between a document and a linked source
  • href tells us the url of the linked page

Git

  • Github is a repository where we can store and access code
  • git checkout -b (branch), creates a new branch
  • git pull origin, pulls from the repository
  • git commit -m, commits changes
  • git add -A, stages the code
  • git checkout -f (branch), ignores changes when leaving a branch

JavaScript

  • The logic of the code
  • A variable is a container which allows us to store the data present in the code
  • Control flow is the order of how a computer will read and execute script
  • JavaScript is case sensitive, always double check the case
  • "Math.random();// random number between 0 and 1Math.floor(Math.random() * 5) + 1; // random integer, from 1 to 5"
  • An array is a collection of elements
